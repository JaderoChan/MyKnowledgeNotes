# [C++] static, volatile, inline, mutable 关键字详解

## static（静态）关键字

- 普通全局变量、`static` 全局变量，全都存储在数据静态区
- 二者内存位置一致，区别只在链接作用域，不在存储区域
- 局部 `static` 变量同样也存放静态区，不会随函数栈销毁

1. 修饰局部变量

   - 生命周期：程序运行全程，函数结束不销毁
   - 作用域：仅限当前作用域
   - 内存：存静态存储区，不占栈空间
   - 只第一次调用初始化，后续调用保留上次值

2. 修饰全局变量/函数

   - 作用域仅限当前源文件，其他文件无法访问
   - 普通全局变量全工程可见，`static` 变成文件私有

3. C++ 修饰类成员

    - 静态成员变量：属于类，所有对象共享一份
    - 静态成员函数：没有 `this` 指针，只能访问静态成员

## volatile（易变）关键字

### 作用

告诉编译器：变量值随时可能被外部修改，不要优化读取，每次都从内存真实读取。

- 不加 `volatile`：编译器会把变量放 CPU 缓存/寄存器，重复读取直接拿缓存值。
- 加 `volatile`：强制每次访问都读写原始内存。

### 适用场景

1. 硬件寄存器（串口、定时器、外设）
2. 多线程共享变量
3. 中断服务程序修改的全局变量

## inline（内联）关键字

### 作用

编译时把函数代码直接嵌入调用处，省去函数压栈、跳转、返回开销。

### 适用场景

代码极短、频繁调用的小函数（几行以内）

### 特性

1. 没有函数调用开销，执行更快
2. 只是建议，编译器可拒绝内联
3. 不能用递归、循环、过多分支

### C++ 额外规则

- 类内直接定义的成员函数默认内联
- 内联函数定义要放头文件，保证全文件一致

## mutable（可变）关键字

### 作用

打破 `const` 成员函数/对象的只读限制。通过 `mutable` 修饰的成员变量可以在 `const` 函数或对象中被修改。

### 适用场景

修饰统计访问次数、缓存标记、临时状态或互斥锁对象，不影响对象核心数据。

### 特性

只能用于 **C++ 类** 的成员变量。
