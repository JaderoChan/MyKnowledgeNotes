# [C++] 类的特殊成员函数的隐式声明与隐式删除规则

## 隐式声明（编译器自动生成）

| 特殊成员函数 | 隐式声明条件                                                 |
| ------------ | ------------------------------------------------------------ |
| **拷贝构造** | 用户未声明：拷贝构造                                         |
| **移动构造** | 用户未声明：拷贝构造、拷贝赋值、移动构造、移动赋值、析构，任何一个都没声明 |
| **拷贝赋值** | 用户未声明：拷贝赋值                                         |
| **移动赋值** | 用户未声明：拷贝构造、拷贝赋值、移动赋值、移动构造、析构，任何一个都没声明 |

> **关键区别**：拷贝操作的隐式生成条件宽松，移动操作的隐式生成条件严格。

## 隐式删除（`= delete`）

即使被隐式声明，以下情况会被定义为 **deleted**：

**拷贝构造/拷贝赋值被隐式删除：**

- 成员或基类的对应拷贝操作是 deleted 或不可访问
- 成员或基类有用户声明的移动操作（C++11起，拷贝构造会被隐式删除，而非不生成——但实践中效果类似）

**移动构造/移动赋值被隐式删除：**

- 成员或基类的对应移动操作是 deleted 或不可访问
- 成员是 `const` 或引用类型（对于移动赋值）

> 注意：移动操作被隐式删除时，不会真正参与重载决议，而是**直接不生成**（回退到拷贝）。这是移动操作的特殊规则，与拷贝不同。

## 简明示例

```cpp
class A
{
    ~A() {}  // 用户声明了析构
    // → 移动构造/移动赋值 不会隐式声明
    // → 拷贝构造/拷贝赋值 仍会隐式声明（但已被C++标准标记为deprecated）
};

class B
{
    B(B&&) {}  // 用户声明了移动构造
    // → 拷贝构造 隐式声明为 deleted
    // → 拷贝赋值 隐式声明为 deleted
    // → 移动赋值 不会隐式声明
};

class C
{
    const int x;
    // → 拷贝赋值/移动赋值 隐式删除（无法赋值const成员）
};
```

## 总结口诀

- **移动操作**：五大函数（拷贝构造、拷贝赋值、移动构造、移动赋值、析构）声明了任意一个，移动就不自动生成
- **拷贝操作**：只有声明了同类拷贝操作才阻止生成（但声明了移动操作会让拷贝 = delete）
- **隐式删除**：成员/基类不支持对应操作时触发
